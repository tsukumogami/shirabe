# PR 19: Auto-install missing tools on command not found

**Author:** @jsmith-contrib
**Status:** Closed (not merged)
**Closed by:** @maintainer-alice
**Closed:** 2025-09-14

## Summary

Hooks into the shell's command-not-found handler. When a command is not found, if
tsuku has a recipe for it, automatically installs it before retrying the command.

## Maintainer comment (closing)

Not building this. The auto-install approach adds significant complexity to tsuku's
startup path and introduces a class of issues where installations happen at unexpected
times — mid-script, inside CI, in environments where the user has no intent to install.

The explicit install model is a deliberate design choice: users should know what they're
installing and when. We've had no requests for auto-install from the community outside
this PR. If demand emerges clearly in the future (multiple independent requests,
not just one PR proposing a solution), we can revisit, but right now this would add
complexity without user benefit.

Closing without merge.
