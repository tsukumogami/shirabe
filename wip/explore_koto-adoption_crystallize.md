# Crystallize Decision: koto-adoption

## Chosen Type
PRD

## Rationale
The exploration revealed a phased, multi-repo effort spanning shirabe skill conversions and koto feature requests. A roadmap would be the ideal fit (multi-feature sequence with dependencies), but it's a deferred type. PRD is the closest available alternative — it captures what to build, the phasing, and the cross-repo dependencies. The PRD can include a phased scope section that functions as the roadmap.

Requirements emerged during exploration (the gap analysis produced specific feature needs that weren't stated upfront). Multiple "stakeholders" need alignment: shirabe skill conversions depend on koto features being built first. The core question is "what should we build" (across two repos) before "how."
